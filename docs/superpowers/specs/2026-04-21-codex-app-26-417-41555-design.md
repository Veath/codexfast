# Codex.app 26.417.41555 Adaptation Design

## Final Scope

Adapt `codexfast` for `Codex.app` `26.417.41555` (`build 1858`) without widening support beyond verified signatures, and preserve compatibility with `26.415.40636` (`build 1799`).

## Final Findings

- The app reports:
  - `CFBundleShortVersionString=26.417.41555`
  - `CFBundleVersion=1858`
- The packed asset layout remains under `app/webview/assets` inside `app.asar`.
- The composer `/fast` target remains in `index-CxBol07n.js`.
- The Settings Fast control moved to `general-settings-D2eks1ok.js`.
- The add-context Speed menu moved to `use-model-settings-ldiRRtPt.js`.
- The Plugins sidebar logic remained in the sidebar/index bundle area and changed more than initially assumed.

## Final Design

### Patch strategy

- Keep the packed `app.asar` workflow unchanged.
- Keep needle-based target discovery unchanged.
- Prefer separate old/new target specs where the bundle drift is meaningful.
- Preserve apply/restore symmetry for every patched path.
- Support normalization of known legacy partial-patch states when they can appear on real user installs.

### Target handling

#### Settings Fast control

- Match `settings.agent.speed.label`.
- Support both old and new helper names in the guarded/patched signatures.
- Preserve the guarded early-return restore shape.

#### Add-context Speed menu

- Match `composer.addContext.speed.option.fast.description`.
- Use separate old/new signatures for:
  - old `Cr()` + `Ir(...)`
  - new `cr()` + `jr(...)`

#### Plugins access

- Match `sidebarElectron.pluginsDisabledTooltip`.
- For `26.417.41555`, patch two related locals in the same sidebar block:
  - disabled Plugins nav item gate: `j = O && A`
  - unified Skills/Plugins nav label gate: `M = D && !A`
- Restore both assignments symmetrically.
- Recognize and normalize the earlier partial-patch shape:
  - `j = !1, M = D && !A`
  - normalize to:
  - `j = !1, M = D`

### Compatibility policy

- Keep the strict whitelist.
- Add `26.417.41555+1858` only with regression coverage and real validation evidence.
- Do not generalize beyond verified build pairs.

## Validation Model

### Regression

The regression suite must cover:

- `26.415.40636` guarded/apply/restore
- `26.417.41555` guarded/apply/restore
- `26.417.41555` partial-Plugins-patch normalization
- unsupported-version blocking before unpack, backup, and re-sign

### Real-app

The real-app validation for this build confirmed:

- Settings Fast control visible
- `/fast` slash command visible
- restore succeeds
- packed `app.asar` layout preserved
- app launches after restart
- `Plugins` sidebar entry visible after the final Plugins fix

## Release Outcome

This adaptation shipped as `codexfast@0.4.1`.

## Non-Goals

- No broad refactor of the embedded patcher.
- No support claim for builds outside the strict whitelist.
