## Bundle Notes

- Assets are still under `app/webview/assets` inside `app.asar`.
- The inspected target files are `general-settings-DobuGNrH.js`, `composer-CUO1FiyC.js`, `check-plugin-availability-C1II8bXB.js`, `use-plugin-install-flow-C0YRtVkW.js`, `use-plugins-BKOuT9XR.js`, and `app-prefetch-impl-CvoY4gd-.js`.
- The Settings-side Fast gate now destructures `isServiceTierAllowed` from the availability hook and calls `xe()` for service-tier settings. The patch removes only the Fast availability guard and preserves `serviceTierSettings.availableOptions.length<=1`.
- The `/fast` slash command is still generated from service-tier command entries in the composer bundle.
- The composer `Intelligence` Speed menu now uses a boolean `_&&serviceTierSettings.availableOptions.length>1` gate. The patch removes only the Fast availability guard and preserves the option-count guard.
- The old Plugins sidebar disabled-nav target, `/skills` page-content auth block, and plugin detail redirect target were not present in the inspected bundle. The shared auth helper in `src-CYrn1fYL.js` allows `authMethod === "apikey"`.
- Curated OpenAI plugin catalog visibility is still hidden for non-ChatGPT auth by `function ge(authMethod){return authMethod !== "chatgpt"}` in `use-plugins-BKOuT9XR.js`; the patch forces this helper to return `false` so the full curated catalog renders instead of the limited-catalog placeholder.
- Plugin install availability still uses the aggregate `connector-unavailable` state in `check-plugin-availability-C1II8bXB.js`.
- Plugin detail app rows are derived from `directoryApps` only in `check-plugin-availability-C1II8bXB.js`; the patch falls back to plugin-declared apps so the detail page still shows the app connect area when the directory app list is unavailable.
- Plugin install modal content now passes `shouldShowInstallDisclosure` into the modal content component; the patch forces that prop off so the basic About/Includes/Capabilities content remains visible.
- The post-install flow closes immediately for `ON_USE` plugins before considering `appsNeedingAuth`; the patch removes that shortcut, falls back to plugin-declared apps when the install response does not return app auth rows, and treats a single pending app like `ON_INSTALL` so the `Connect <App>` permission modal opens after installation.
- Startup plugin prefetch now requests the remote `shared-with-me` marketplace in `app-prefetch-impl-CvoY4gd-.js`; the patch removes that remote marketplace prefetch while preserving workspace-directory prefetch.
