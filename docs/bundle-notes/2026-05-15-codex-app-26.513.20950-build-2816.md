## Bundle Notes

- Assets are still under `app/webview/assets` inside `app.asar`.
- At runtime, the same renderer assets are requested as `app://-/assets/*.js`. Runtime launch must intercept both this current URL shape and the legacy `app://-/webview/assets/*.js` shape.
- Runtime launch must keep the CDP Fetch session open for the lifetime of the launched Codex process. `general-settings-Bvwhh0-i.js` and `skills-page-ydcMfNsx.js` are lazy-loaded after opening Settings or Plugins, so a launch process that exits after the initial window cannot patch Settings Fast or Plugins page content.
- The runtime launch session uses a CDP heartbeat and at most three reconnect attempts. Reconnect re-enables `Page` and `Fetch` and reloads the renderer; exhausted reconnects print `Runtime patch session lost`. After a successful startup, Codex keeps running if the patch session is lost, but later lazy-loaded chunks can no longer be patched by that launch process.
- The inspected target files are `general-settings-Bvwhh0-i.js`, `composer-CL8HPtlL.js`, `app-main-Dsg36Y4q.js`, `skills-page-ydcMfNsx.js`, `plugin-detail-page-CQ7HXo6z.js`, `check-plugin-availability-Cl1_8Dsf.js`, `plugins-availability-Ch97MI4l.js`, and `prosemirror-DbRBBq50.js`.
- The Settings-side Fast gate now uses `se()` for availability and `fe()` for the service-tier hook.
- The Fast slash command remains in a `composer-*.js` asset and still uses the existing `enabled:<local>` target shape.
- The composer `Intelligence` Speed submenu now renders through an inline `ve=v?...:null` assignment rather than a separate `va(...)` local.
- The Plugins sidebar disabled-nav state now uses `Is("533078438")` with `Ml(authMethod)`.
- The unified Skills/Plugins nav label state now uses `hl({hostId:Io})` and an API-key exclusion.
- The `/skills` route page-content gate still matches the existing API-key-derived local assignment shape.
- The plugin detail redirect now uses `if(ue(i))` after `authMethod` is read.
- The plugin install availability gate moved out of `plugins-availability-*.js` into `check-plugin-availability-Cl1_8Dsf.js`.
- The plugin install modal content gate remains in `plugins-availability-Ch97MI4l.js`.
- The composer `@` plugin mention list calls the plugin list hook with `additionalMarketplaceKinds:[\`shared-with-me\`]`, which triggers the remote plugin catalog that requires ChatGPT authentication. The patch changes this additional marketplace list to `[]`; it does not add remote ChatGPT catalog support.
- The model-list bridge still delegates to `model/list`, but `codexfast` skips GPT-5.5 apply targets on this build because GPT-5.5 is expected to be visible through the official app path from `26.422.30944` onward.
